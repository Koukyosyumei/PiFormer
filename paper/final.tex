\begin{abstract}
%% ============================================================
Verifiable inference for transformers requires the prover to convince a
sceptic that a specific committed model produced a specific output on a
public input. The dominant cost in such proofs comes from
\emph{ZK-unfriendly} primitives in the standard transformer stack:
softmax (exponentiation, division, $\max$), GeLU (transcendental
activation), Layer Normalization (square root and division), and dense
matrix multiplication (one field multiplication per weight entry).

We present \underline{\sys}, a transformer architecture and a matching
SNARK prover that are \emph{co-designed} for verifiable inference. The
architecture has three pieces. \emph{Learnable Lookup Attention}~(LLA)
replaces softmax with a kernel attention $\bphi(\bQ)\bigl(\bphi(\bK)^\top
\bV\bigr)$ whose feature map decomposes additively into small lookup
tables that fit Lasso natively. \emph{Ternary projections} restrict all
weight matrices to $\{-1,0,+1\}$ scaled by a single learnable
$\alpha$, eliminating field multiplications inside the contraction.
\emph{Constraint-fused LayerNorm} expresses both the variance and the
output identity as low-degree sumcheck claims with chunked range
proofs, removing every division and square root from the circuit.

The prover combines Hyrax PCS (transparent, BN254), the Spartan
sumcheck IOP, and Lasso lookups, but its key engineering innovation is
\emph{cross-block batching}. Six per-protocol sumchecks are run once
across all $L$ blocks; all $\Phi(\bQ)$/$\Phi(\bK)$ Lasso instances
across all blocks fold into one global Lasso; all LayerNorm range
proofs share one multiplicity commitment; residuals are checked
homomorphically; and all final Hyrax openings flow through deferred
batch accumulators that finalise with one MSM pair per group. We
implement the full pipeline in $\sim$10\,k lines of Rust and run an
end-to-end smoke test that produces a 108\,KB transparent proof for a
1-block model in 0.27\,s on a single CPU core.
\end{abstract}

\section{Introduction}

ZK-SNARKs are the natural primitive for verifiable machine learning:
the model owner convinces a verifier that a claimed output came from a
specific frozen model, while the verifier never reads the
weights~\cite{liu2021zkcnn,sun2024zkllm,abbaszadeh2024zero}. Yet
proving a transformer~\cite{vaswani2017attention} block in a SNARK is
expensive precisely because the operations that make transformers
trainable (softmax, GeLU, LayerNorm) are exactly the operations that
finite-field circuits handle poorly. Recent
work~\cite{sun2024zkllm,qu2025zkgpt} retrofits ZK proofs onto
pre-trained softmax transformers and pays a high concrete cost in
both prover time and proof size.

\sys{} starts from a different premise: \emph{co-design} the model and
the proof system so that every operation in inference maps to a
low-degree sumcheck or a structured table lookup. The three modelling
ingredients are (i) \emph{Learnable Lookup Attention}, a linear
attention~\cite{katharopoulos2020transformers,wang2020linformer} whose
feature map is a sum of learnable sub-tables sized to fit Lasso
exactly; (ii) \emph{ternary
projections}~\cite{ma2025bitnet} that absorb all magnitude into a
single per-layer scale and replace field multiplications with field
additions; and (iii) a \emph{division-free LayerNorm} circuit that
proves the normalisation identity at one random point via constraint
fusion plus chunked Lasso range proofs.

The proof system itself is built around \emph{cross-block batching}:
six protocol-typed sumchecks (QKV projection, output projection,
attention-output, attention-context, FFN-Y, FFN-M) and one global
Lasso are each run \emph{once} across all $L$ blocks, with per-block
claims combined by Fiat-Shamir powers of $\eta$.  Residuals are
verified by homomorphic addition of Hyrax commitments.  All Hyrax
openings are aggregated through deferred batch accumulators that
finalise into one MSM pair per group.  These are not
soundness-affecting tricks: each is a standard random-linear-combination
argument whose error is dominated by the inner sumcheck error.

\paragraph{Contributions.}
\begin{itemize}
  \item \textbf{LLA + ternary + ZK-friendly LayerNorm.}
        A transformer block in which every operation, including
        attention, normalisation, and projection, has a sub-linear
        verifier circuit (\S\ref{sec:arch}).
  \item \textbf{Cross-block batched protocol.}
        An end-to-end prover that collapses six per-block
        protocols into six \emph{model-level} batched sumchecks,
        plus one global Lasso and one global range batch
        (\S\ref{sec:protocol}).
  \item \textbf{Implementation.}
        $\sim$10\,k lines of Rust over BN254, transparent setup,
        single-core CPU, with a working CLI
        (\texttt{piformer setup}/\texttt{prove}/\texttt{verify})
        and an end-to-end smoke test that produces a
        108\,KB proof for a 1-block model in 0.27\,s
        (\S\ref{sec:eval}).
\end{itemize}

\section{Background}
\label{sec:bg}

\paragraph{Transformer block.}
Each block computes
\begin{align*}
  \bX_{\mathrm{mid}} &= \bX_{\mathrm{in}} + \Att\bigl(\LN(\bX_{\mathrm{in}})\bigr),
  \\
  \bX_{\mathrm{out}} &= \bX_{\mathrm{mid}} + \mathrm{FFN}\bigl(\LN(\bX_{\mathrm{mid}})\bigr),
\end{align*}
where standard self-attention is
$\Att(\bQ,\bK,\bV) = \softmax(\bQ\bK^\top/\sqrt{d})\bV$
\cite{vaswani2017attention}, and LayerNorm
$\LN(\bx) = \bgamma\odot(\bx-\mu)/\sqrt{\sigma^2+\epsilon} + \bbeta$
\cite{ba2016layer}.  Each of softmax, GeLU, and LayerNorm contains
operations (exp, division, square root) without compact polynomial
representations, so they dominate SNARK complexity.

\paragraph{Multilinear extensions and sumcheck.}
A multilinear polynomial $\MLE : \FF^n \to \FF$ is determined by its
$2^n$ evaluations on $\{0,1\}^n$. The sumcheck
protocol~\cite{setty2020spartan} reduces a claim
$H = \sum_{x\in\{0,1\}^n} f(x)\cdot g(x)$ to a single evaluation
$f(r)\cdot g(r)$ at a random $r\in\FF^n$, with $O(n)$ rounds of
constant-degree round polynomials.

\paragraph{Hyrax PCS.}
For a multilinear polynomial of $2^n=2^{\nu+\sigma}$ evaluations
arranged as a $2^\nu\!\times\!2^\sigma$ matrix $M$, Hyrax commits to
each row as a vector Pedersen commitment
$C_i = \mathrm{MSM}(\mathbf{g}, M[i])$ and opens at
$r = (r_L\Vert r_R)$ by sending the row-collapsed vector
$w'_j = \sum_i L_i(r_L)\cdot M[i][j]$.  The verifier checks both a
homomorphic commitment equation and an inner-product
equation~\cite{wahby2018doubly}.  Hyrax is transparent (no trusted
setup) under the discrete-log assumption in BN254 G1; commitments are
$O(\sqrt N)$ in size, and openings cost $O(\sqrt N)$ G1 operations.

\paragraph{Lasso lookups.}
For a sub-table $T\in\FF^{2^m}$ and queries
$\{(\mathrm{idx}_j, v_j)\}$ with $v_j = T[\mathrm{chunk}(\mathrm{idx}_j)]$,
Lasso~\cite{setty2024unlocking,arun2024jolt} proves all queries are
consistent with $T$ via a single batched
sumcheck for
$\sum_x \widetilde T(x)\cdot L(x) = \sum_j \rho^j\,T[\mathrm{ch}_j]$
where $L(x)=\sum_j \rho^j\,\widetilde\eq(\mathrm{bin}(\mathrm{ch}_j),x)$
and $\rho$ is Fiat--Shamir-derived. A composite table that decomposes
additively into $c$ sub-tables (the canonical Lasso ``structured''
condition) is committed as $c$ small tables of size $2^{\bits/c}$
instead of one monolithic table of size $2^\bits$.

\section{The \sys{} Architecture}
\label{sec:arch}

\subsection{Block structure}
A \sys{} block is a pre-norm linear-attention block:
\begin{align}
  \bX_{\mathrm{n1}} &= \LN(\bX_{\mathrm{in}}; \bgamma_1, \bbeta_1),
  \\
  \bQ,\bK,\bV   &= \bX_{\mathrm{n1}}\bW_{Q,K,V}
                   \quad\text{(ternary projections, shared sumcheck)},
  \\
  \bO_{\mathrm{attn}} &= \LLA(\bQ,\bK,\bV)\,\bW_O,
  \\
  \bX_{\mathrm{mid}} &= \bX_{\mathrm{in}} + \bO_{\mathrm{attn}}
                       \quad\text{(homomorphic residual)},
  \\
  \bX_{\mathrm{n2}} &= \LN(\bX_{\mathrm{mid}}; \bgamma_2, \bbeta_2),
  \\
  \bO_{\mathrm{ffn}} &= \bphi(\bX_{\mathrm{n2}}\bW_1)\,\bW_2,
  \\
  \bX_{\mathrm{out}} &= \bX_{\mathrm{mid}} + \bO_{\mathrm{ffn}}.
\end{align}
After $L$ such blocks the model applies a final LayerNorm and a
ternary language-model head.

\subsection{Learnable Lookup Attention}
\label{ssec:LLA}

Standard softmax attention requires $n^2$ exponentials and $n$
row-wise divisions; both translate poorly to SNARK circuits.  We
replace it with a positive-definite kernel
$\kappa(\bq,\bk)=\langle\bphi(\bq),\bphi(\bk)\rangle$ where
$\bphi:\RR^{d_h}\!\to\RR^{d_h}$ is applied element-wise:
\begin{equation}
  \LLA(\bQ,\bK,\bV)_t \;=\;
  \frac{\bphi(\bq_t)^{\!\top}\!\bigl(\sum_{s=1}^{n}\bphi(\bk_s)\bv_s^{\!\top}\bigr)}
       {\bphi(\bq_t)^{\!\top}\!\bigl(\sum_{s=1}^n\bphi(\bk_s)\bigr)}.
  \label{eq:LLA}
\end{equation}
The context matrix $\bM=\sum_s \bphi(\bk_s)\bv_s^\top \in
\RR^{d_h\times d_h}$ depends only on the keys/values and is computed
once: per-token output costs $O(d_h^2)$ and the per-block prover cost
becomes $O(nd_h^2)$ instead of $O(n^2 d_h)$. The denominator is
computed for training stability~\cite{katharopoulos2020transformers};
the SNARK currently proves the un-normalised numerator (\S\ref{sec:eval}).

\paragraph{Additive sub-table feature map.}
Rather than a fixed kernel (e.g., $\mathrm{elu}+1$), \sys{}
parametrises $\bphi$ as a sum of learnable sub-table lookups. Let
$\bits$ be the input bit-width and $\chunks\mid\bits$ the
decomposition depth, $\chunkbits=\bits/\chunks$ bits per chunk.
For scalar $x$, define the integer index
\begin{equation}
  x_{\mathrm{int}} = \mathrm{clamp}\bigl(\lfloor x/s\rfloor,\;0,\;2^\bits-1\bigr),
\end{equation}
and chunk extractor $\chi_i(x_{\mathrm{int}})
= (x_{\mathrm{int}} \gg i\chunkbits)\,\&\,(2^{\chunkbits}-1)$.
Then
\begin{equation}
  \bphi(x) \;=\; \sum_{i=0}^{\chunks-1} T_i\bigl[\chi_i(x_{\mathrm{int}})\bigr],
  \label{eq:phi}
\end{equation}
where $T_0,\dots,T_{\chunks-1}\in\RR^{2^{\chunkbits}}$ are
learnable. The sub-tables are initialised so $\sum_i T_i$ approximates
GeLU and trained end-to-end with the straight-through
estimator~\cite{bengio2013estimating}; gradients flow to the table
entries while the quantisation/round step is bypassed.
For $\bits=16, \chunks=2$, equation~\eqref{eq:phi} replaces a
monolithic 65{,}536-entry table with two 256-entry tables, exactly
matching the Lasso ``structured table'' condition: a single query
$x_{\mathrm{int}}$ to the implicit $2^\bits$-table decomposes into
$\chunks$ independent sub-table queries whose outputs are summed.

\subsection{Ternary projections}
\label{ssec:ternary}

All projection matrices $\bW_Q,\bW_K,\bW_V,\bW_O,\bW_1,\bW_2$ and
the LM head are constrained to entries in $\calT=\{-1,0,+1\}$ scaled
by a single learnable scalar $\alpha\in\FF$ per layer, following the
BitNet b1.58 design~\cite{ma2025bitnet}.  Forward pass:
$\by = \alpha\cdot(\bx\bW)+\bb$ with $\bW\in\calT^{m\times n}$.
Training uses STE with the magnitude-thresholded mapping
$w\mapsto \mathrm{sign}(w)\cdot[\,|w|>0.7\overline{|w|}\,]$;
$\alpha$ is multiplied in \emph{after} the matrix product so the
in-circuit weight stays ternary.

In the SNARK, a sumcheck for a ternary contraction
$Y(r_t,r_d)-b(r_d) = \alpha\sum_k X(r_t,k)\,W(k,r_d)$
materialises only the cheap vector $\alpha\cdot W(\cdot,r_d)$, never
a field weight matrix; the prover folds the equality polynomial
against ternary entries with $+\!=$/$-\!=$/skip rather than field
multiplications.  A stand-alone Lasso check against the 4-element
table $[0,1,p-1,0]$ enforces ternarity and is a candidate to fold
into setup-time preprocessing, so the verifying key carries a binding
proof that $\bW\in\calT^{m\times n}$.

\subsection{Constraint-fused LayerNorm}
\label{ssec:ln}

Let $d$ be the embedding dimension, $\bx\in\ZZ^d$ a row,
$\mathrm{sum\_x}=\sum_j x_j$, $\mathrm{var\_x}=\sum_j(d\cdot x_j-\mathrm{sum\_x})^2$
the integer numerator, and $\sigma=\lfloor\sqrt{\mathrm{var\_x}}/d\rfloor$.
\sys{} proves the entire LayerNorm at a single random row index
$r_t\in\FF^{t_{\mathrm{bits}}}$ in four steps, with no field divisions:
\begin{enumerate}
  \item \textbf{Mean sumcheck.}
        Degree-2 sumcheck for
        $\widetilde{\mathrm{sum\_x}}(r_t) = \sum_j x_{\mathrm{col}}(j)$
        where $x_{\mathrm{col}} = \widetilde\bx.\mathrm{fix\_row}(r_t)$.
  \item \textbf{Variance sumcheck.}
        Cubic sumcheck for
        $\widetilde{\mathrm{var\_x}}(r_t) = \sum_j h(j)^2$
        with $h(j) = d\cdot x_{\mathrm{col}}(j) - \widetilde{\mathrm{sum\_x}}(r_t)$.
  \item \textbf{Sigma range proof.}
        Show $\sigma$ is the integer floor square-root by range-checking
        $\mathrm{var\_x} - (d\sigma)^2 \ge 0$ and
        $(d\sigma+d)^2 - 1 - \mathrm{var\_x} \ge 0$.
        Both residuals go through a chunked Lasso range proof
        against the identity table $T_{\mathrm{id}}[i]=i$ of size
        $2^{16}$ (see \S\ref{ssec:cross-block}).
  \item \textbf{Y constraint fusion.}
        For a single random $(r_{y_t}, r_{y_d})$, audit
        $\bgamma\odot(d\bx-\mathrm{sum\_x}) + \bbeta\cdot d\sigma
        \approx d\sigma\cdot \by$
        (up to integer rounding) by another range proof on the
        $\mathrm{lo}/\mathrm{hi}$ residuals.  The $\gamma X$ and
        $\sigma Y$ legs are fused into a single multi-batched cubic
        sumcheck.
\end{enumerate}
The verifier evaluates $\bgamma$ and $\bbeta$ directly from the VK
in $O(d)$ field operations per layer; everything else is one shared
opening per witness.

\section{End-to-End Protocol}
\label{sec:protocol}

\subsection{Preprocessing}
\textsf{Setup} commits each ternary weight matrix and each activation
sub-table via Hyrax. The proving key contains the raw ternary arrays
plus their commitments; the verifying key contains commitments only
(no raw weights).  Hyrax generators are derived as
$g_j=\mathrm{SHA3}(\texttt{"piformer-hyrax-gen"}\Vert j)\cdot G$,
giving a fully transparent setup.

\subsection{Per-block Phase 1}
The only truly per-block step is to commit seven intermediate matrices
($\bX_{\mathrm{n1}}, \bQ, \bK, \bV, \bO_{\mathrm{attn}},
\bX_{\mathrm{n2}}, \bO_{\mathrm{ffn}}$), prove both LayerNorms, and
fold both LayerNorms' four range witnesses
($\sigma_{\mathrm{LN1}}, y_{\mathrm{LN1}},
  \sigma_{\mathrm{LN2}}, y_{\mathrm{LN2}}$) into one shared
multiplicity commitment.  Residuals are
\emph{not} proved: the verifier uses the linearity of Hyrax to compute
$C_{X_{\mathrm{mid}}}=C_{X_{\mathrm{in}}}+C_{O_{\mathrm{attn}}}$ and
$C_{X_{\mathrm{out}}}=C_{X_{\mathrm{mid}}}+C_{O_{\mathrm{ffn}}}$
locally.  Phase 1 produces no projection, attention, or FFN sub-proof;
those are deferred to model-level batches below.

\subsection{Six cross-block batched sumchecks}
\label{ssec:cross-block}

After all $L$ Phase-1 commitments are absorbed, the prover draws a
\emph{single} global random point $r_{td}=(r_t\Vert r_{\mathrm{out}})$
from the transcript and runs six \emph{model-level} sumchecks. Each
batches the analogous per-block claim across all $L$ blocks, combined
by Fiat--Shamir powers of an independent challenge $\eta$:

\medskip
\begin{tabular}{lll}
  \toprule
  \textbf{Batch} & \textbf{Statement} & \textbf{Shared challenge}\\
  \midrule
  \texttt{batch\_qkv}      & $\bQ,\bK,\bV = \bX_{\mathrm{n1}}\bW_{Q,K,V}$ (per block)
                           & $r_k^{qkv}$ \\
  \texttt{batch\_oproj}    & $\bO_{\mathrm{attn}} = (\bphi(\bQ)\bphi(\bK)^\top\bV)\bW_O$
                           & $r_k^o$ \\
  \texttt{batch\_attn\_out}& $\mathrm{out}[t,j] = \sum_i \bphi(\bQ)[t,i]\,\bM[i,j]$
                           & $r_{\mathrm{out}}^{\mathrm{attn}}$ \\
  \texttt{batch\_attn\_ctx}& $\bM[i,j] = \sum_t \bphi(\bK)[t,i]\,\bV[t,j]$
                           & $r_{\mathrm{ctx}}^{\mathrm{attn}}$ \\
  \texttt{batch\_ffn\_y}   & $\bY_{\mathrm{ffn}} = \bA\bW_2$
                           & $r_k^{fy}$ \\
  \texttt{batch\_ffn\_m}   & $\bM_{\mathrm{ffn}} = \bX_{\mathrm{n2}}\bW_1$
                           & $r_k^m$ \\
  \bottomrule
\end{tabular}
\medskip

A single sumcheck-multi protocol
($H = \sum_\ell \eta^{\ell-1}\sum_x f_\ell(x)g_\ell(x)$)
turns $L$ separate sumcheck transcripts into one of length
$\log\,(\text{contraction dim})$.  Soundness is preserved because
$\eta$ is drawn \emph{after} all per-block claims and commitments are
absorbed into the transcript: a single false claim contributes a
non-zero polynomial in $\eta$ of degree $\le L-1$, caught by a random
$\eta$ except with probability $(L-1)/|\FF|$.

\subsection{Global Lasso and global range batches}
All $\bphi(\bQ)$ and $\bphi(\bK)$ Lasso instances across all blocks
fold into one \texttt{LassoMultiProof} via a single combined sumcheck
and one Hyrax opening.  Similarly, all six LayerNorm range witnesses
in a block (four per block plus two for the final LN) share a
\emph{single} multiplicity commitment $m_{\mathrm{com}}$ against the
identity table $T_{\mathrm{id}}[i]=i$ of size $2^{16}$, replacing
five $\sqrt{2^{16}}$-MSM commitments per block with one.

\subsection{Global intermediate batch open + deferred accumulators}
After all six cross-block sumchecks have fixed
$r_{td}=(r_t\Vert r_{\mathrm{out}})$, a \emph{single}
\texttt{HyraxOpenBatch} opens the $5L$ intermediate matrices
($\bQ_\ell, \bK_\ell, \bV_\ell, \bO_{\mathrm{attn}}^\ell,
\bO_{\mathrm{ffn}}^\ell$) at the shared point $r_{td}$, using one
deferred MSM pair instead of $5L$.  The verifier maintains ten
cross-cutting accumulators (LayerNorm row/td, projection W/b, LM-head
W/b, range $\sigma$/Y/$m$, intermediate) into which every Hyrax claim
is pushed; each accumulator finalises once at the end of verification
with a single MSM pair (deferred-$\mu$ challenge for the
inner-product check, single MSM for the homomorphic check).

\subsection{Final stage and verification}
The model proof concludes with a final LayerNorm + LM head, also
batched into the same accumulators.  The verifier (i) re-commits the
public input/output and binds them to the proof's $x_{\mathrm{in\_com}}$
and $\mathrm{logits\_com}$, (ii) walks all $L$ blocks' Phase 1, (iii)
verifies the six cross-block sumchecks, (iv) verifies the final LN +
LM head, (v) verifies the global intermediate batch open and 17
matrix-type batch opens, (vi) finalises the ten Hyrax accumulators,
and (vii) verifies the global Lasso.  Total verifier work is
$O(L\cdot\sqrt{Td})$ G1 operations and $O(L\log Td)$ field operations.

\subsection{Soundness, briefly}
\label{ssec:sound}

\begin{theorem}[\sys{} soundness, informal]
Under the discrete-log assumption on BN254 G1, the soundness error of
the full \sys{} prover is at most
$L\cdot n_{\max}\cdot \delta_{\max}/|\FF| + \varepsilon_{\mathrm{DL}}$,
where $n_{\max}$ is the maximum number of sumcheck variables across
all sub-protocols, $\delta_{\max}\le 3$ is the maximum round
polynomial degree, $L$ is the number of blocks, and
$\varepsilon_{\mathrm{DL}}$ is the DL advantage in BN254 G1.
\end{theorem}

The proof proceeds by hybrid over the $L$ blocks. Hyrax binding
(Wahby et al.~\cite{wahby2018doubly}) reduces all forged commitments
to DL. Each cross-block batched sumcheck is sound because the
combining challenge $\eta$ is drawn \emph{after} all per-block claims
are absorbed (RLC argument over a polynomial of degree $\le L-1$ in
$\eta$ plus the inner sumcheck error). Lasso soundness follows from
the structured-table sumcheck plus Hyrax binding on the
sub-tables~\cite{setty2024unlocking}.  Range-proof soundness uses the
LogUp identity over a globally committed multiplicity vector.
Homomorphic residuals require no proof at all: linearity of MSM gives
$\Com(A)+\Com(B)=\Com(A+B)$ for free. For $L=12$, $n_{\max}=64$,
$\delta_{\max}=3$ over BN254, the statistical error is $\le 2^{-242}$,
dominated by the DL term.

\section{Implementation and Evaluation}
\label{sec:eval}

\sys{} is implemented in $\sim$10\,k lines of Rust over the BN254
scalar field, plus a PyTorch training pipeline that exports
field-quantised weights and witnesses so the integer arithmetic in
Python and the Rust circuit match exactly.  The CLI exposes
\texttt{piformer setup}/\texttt{prove}/\texttt{verify}/\texttt{inspect}/\texttt{sample}.
The current prover supports single-head attention; multi-head is on
the roadmap but does not change the protocol structure.

\paragraph{Smoke-test benchmark.}
We measure the end-to-end CLI on a randomly initialised model exported
from PyTorch.  Setup, prove, and verify times are the inner
\emph{compute}-only timings reported by the Rust CLI itself (i.e., the
preprocessing, \texttt{prove}, and \texttt{verify} calls), excluding
JSON parsing and proving/verifying-key file IO.  All numbers are
single-thread wall-clock on a single x86\_64 CPU core; proof sizes are
the on-disk size of the binary proof bundle.

\medskip
\begin{tabular}{lrrrrrrrrr}
  \toprule

The prove time scales roughly linearly in the number of intermediate
matrix entries ($L\!\cdot\!T\!\cdot\!d$ and $L\!\cdot\!T\!\cdot\!d_{ff}$):
the per-block sumchecks are $O(Td)$/$O(Td_{ff})$ and the cross-block
batching collapses transcript work but not arithmetic work. Verifier
compute is sub-linear in the model size because every cross-block
sumcheck is a $\log Td$-round transcript and the deferred Hyrax
accumulators finalise in one MSM pair per group (vs.\ $O(L)$ pairs).
For the \texttt{medium} configuration ($\sim$924\,k parameters), pure
verification takes 0.28\,s while the surrounding JSON parsing and
file IO add another 0.7\,s of wall-clock for a total CLI invocation
of $\sim$1\,s; this gap is engineering, not protocol.

Larger configurations (\texttt{large}: $d{=}256$, $L{=}6$;
\texttt{gpt2-small}: $d{=}512$, $L{=}12$) are implemented and pass
the same setup/prove/verify cycle, but the JSON witness export
becomes the dominant time at that end (170\,MB witness for the
\texttt{large} configuration); they are excluded from the table
because their wall-clock is no longer dominated by the prover. A
full side-by-side comparison with zkLLM~\cite{sun2024zkllm} and
zkGPT~\cite{qu2025zkgpt} requires fixing matched architectures and
matched quantisation, and is left to future work.

\paragraph{Status of training.}
The Python pipeline (custom \texttt{LinearAttentionLayer},
\texttt{StructuredLookupActivation}, \texttt{TernaryLinear}) trains
end-to-end with STE; we do not yet report perplexity vs.\ a softmax
baseline. The integer/field forward pass used for the witness is
identical to the Rust circuit's arithmetic, eliminating the
train--prove gap.

\section{Discussion and Limitations}

\paragraph{What is and is not in the SNARK.}
The Rust prover proves the un-normalised LLA output
$\bphi(\bQ)(\bphi(\bK)^\top\bV)\bW_O$. The Python forward pass adds
the row-wise normaliser $Z_t=\bphi(\bq_t)^\top\sum_s\bphi(\bk_s)$ for
training stability; matching the two requires either disabling $Z$ at
export time or extending the circuit with a row-normalisation
sumcheck.  The current implementation also lacks causal
masking, multi-head attention, and zero-knowledge masking.  None
changes the protocol architecture: causal masking is a sumcheck-time
indicator, multi-head splits the attention sumchecks into parallel
batches, and ZK is an additional blinding layer over committed
witnesses.

\paragraph{From benchmark to deployment.}
Our timings are CPU and single-threaded.  GPU MSM and parallel
sumcheck~\cite{qu2025zkgpt} would translate directly to \sys{}
because every dominant inner loop (cross-block batched sumcheck,
Lasso multi-instance, batched Hyrax open) is data-parallel.
Streaming generation is the natural fit for Nova-style
folding~\cite{kothapalli2022nova}: the running context matrix
$\bM=\bphi(\bK)^\top\bV$ is the IVC accumulator, and one folding step
per token yields constant-time amortised proving.

\paragraph{Beyond \sys{}.}
The two stand-alone Rust modules
(\texttt{attention/ternary\_check.rs} and
\texttt{cross\_layer/projection.rs}) implement (i) an in-circuit
ternary-weight check via a 4-element Lasso table $[0,1,p-1,0]$ and
(ii) a single cubic sumcheck collapsing $L$ per-layer projections
into one $\log L + \log d_{\mathrm{in}}$-round sumcheck.  Folding both
into the end-to-end prover gives a setup-time binding proof on the
weights and a further factor of $L$ saving on the projection
transcript.  Together with an EVM Hyrax verifier and a
zero-knowledge masking layer, this lays out the path from a working
research prototype to an on-chain verifiable inference service.
